# デザインレビュースキル

**スキル名**: `design-review`

## 概要

フロントエンドのデザインを「AI Slop」美学から守り、独創的で印象的なUIを確保するためのレビューを行います。

## 使用タイミング

- **新規UIコンポーネント作成時**
- **デザインシステム構築時**
- **カラーパレット選定時**
- **タイポグラフィ選定時**
- **アニメーション実装時**

## デザイン原則

**「AI Slop」美学を避け、独創的で印象的なフロントエンドを作成する**

## レビュー観点

### 1. タイポグラフィ

**避けるべきフォント:**
- Inter, Roboto, Arial, System fonts

**推奨:**
- Google Fontsから独特で美しいフォントを選択
- コンテキストに合った個性的なフォント

### 2. カラー & テーマ

**避けるべき配色:**
- 紫のグラデーション on 白背景
- 青→紫のグラデーション

**推奨:**
- 支配的な色と鋭いアクセントカラー
- CSS変数で一貫性を保つ
- 大胆で独自の配色

### 3. モーション

**推奨:**
- 高インパクトな瞬間に集中（ページロード、staggered reveals）
- CSS-onlyソリューションを優先
- `animation-delay`でオーケストレーション

**避けるべき:**
- 散在するマイクロインタラクション
- 過剰なアニメーション

### 4. 背景

**推奨:**
- CSS gradientを重ねる
- 幾何学的パターン
- 雰囲気と深みを作る

**避けるべき:**
- 単色の背景のみ
- 予測可能なパターン

## デザインレビューチェックリスト

- [ ] **タイポグラフィ**: Inter, Roboto, Arialを避けているか
- [ ] **カラー**: 紫のグラデーションを避けているか
- [ ] **カラー**: CSS変数で色を管理しているか
- [ ] **モーション**: アニメーションが戦略的に配置されているか
- [ ] **背景**: 単色だけでなくグラデーションやパターンを使用しているか
- [ ] **全体**: 独創的で印象的なデザインか

## 使用例

```bash
/design-review
/design-review components/ui/
/design-review app/globals.css
```
