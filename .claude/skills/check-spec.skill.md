# 仕様書確認スキル

**スキル名**: `check-spec`

## 概要

実装が仕様書（`docs/codepen_html.md`）から逸脱していないかを確認し、仕様との整合性を検証します。

## 使用タイミング

1. **作業開始前**: 仕様書を読み、実装する機能が記載されているか確認
2. **各フェーズ完了後**: 仕様書と実装内容を照合
3. **コミット前**: 仕様通りに実装されているか最終確認

## 仕様書の役割

`docs/codepen_html.md` はプロジェクトの**唯一の真実の源（Single Source of Truth）**です。

すべての実装判断はこの仕様書に基づいて行われなければなりません。

## 確認プロセス

### ステップ1: 仕様書の読み込み

```bash
Read docs/codepen_html.md
```

### ステップ2: 実装内容の照合

**1. 機能の完全性**
- [ ] 仕様書に記載された機能がすべて実装されているか
- [ ] 仕様書に記載のない機能を追加していないか

**2. データ構造の一致**
- [ ] 型定義が仕様書と一致しているか
- [ ] データベーススキーマが仕様書通りか

**3. ファイル構造の一致**
- [ ] ディレクトリ構造が仕様書通りか
- [ ] ファイル命名規則が仕様書通りか

**4. 実装方法の一致**
- [ ] 指定されたライブラリを使用しているか
- [ ] セキュリティ対策が仕様書通りか

### ステップ3: 逸脱の記録

1. **軽微な逸脱**: ユーザーに報告して確認
2. **重大な逸脱**: 即座に修正
3. **意図的な逸脱**: 仕様書を更新

### ステップ4: IMPLEMENTATION_STATUS.md の更新

照合結果を記録:

```markdown
## [フェーズ名]
- [x] 仕様書との照合完了
- [x] 逸脱なし
```

## 実装の原則

1. **仕様書厳守**: `codepen_html.md`から逸脱しない
2. **段階的実装**: フェーズごとに照合
3. **コミット戦略**: フェーズごとにコミット

## チェックリスト

### 実装前
- [ ] `docs/codepen_html.md`を読んだ
- [ ] 実装する機能が仕様書に記載されているか確認した

### 実装後
- [ ] 仕様書と実装内容を照合した
- [ ] 仕様からの逸脱がないことを確認した
- [ ] `docs/IMPLEMENTATION_STATUS.md`を更新した

## 使用例

```bash
/check-spec
/check-spec phase-1
/check-spec authentication
```
